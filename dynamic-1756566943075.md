## 1. Programação Orientada a Objetos (POO)

A POO é fundamental para criar códigos mais organizados, reutilizáveis e de fácil manutenção. Python oferece uma implementação completa de POO, incluindo classes, herança, encapsulamento e polimorfismo.

### 1.1 Classes e Objetos

**Definição de uma classe:**
```python
class Animal:
    def __init__(self, nome):
        self.nome = nome

    def emitir_som(self):
        print(f"{self.nome} faz um som.")
```

**Criando objetos:**
```python
 cachorro = Animal("Rex")
 cachorro.emitir_som()  # Saída: Rex faz um som.
```

### 1.2 Herança e Polimorfismo

**Herança:**
```python
class Cachorro(Animal):
    def emitir_som(self):
        print(f"{self.nome} late.")
```

**Polimorfismo:**
```python
animais = [Animal("Gato"), Cachorro("Rex")]
for animal in animais:
    animal.emitir_som()
# Saída:
# Gato faz um som.
# Rex late.
```

### Diagrama explicativo de classes e herança:

```
        +------------------+
        |     Animal       |
        +------------------+
        | - nome: str     |
        +------------------+
        | emitir_som()     |
        +------------------+
               /   \
              /     \
   +----------------+  +-------------------+
   |   Gato         |  |   Cachorro        |
   +----------------+  +-------------------+
   | emitir_som()   |  | emitir_som()      |
   +----------------+  +-------------------+
```

## Manipulação Avançada de Arquivos e Bancos de Dados em Python

### Manipulação de Arquivos: Leitura, Escrita, Atualização e Exclusão

Python oferece várias formas de manipular arquivos de forma eficiente e segura, usando as funções integradas e módulos especializados. Vamos explorar operações avançadas com exemplos práticos.

#### 1. Leitura de Arquivos (Modo 'rb', 'rt' com grandes volumes)

Para processar grandes volumes de dados, é recomendado ler o arquivo em partes, usando buffers ou leitura linha a linha:

```python
with open('large_file.txt', 'r', encoding='utf-8') as arquivo:
    for linha in arquivo:
        processa(linha)  # substitua pela sua lógica de processamento
```

#### 2. Escrita (Incluindo Append e sobrescrever)

Para evitar perder dados, o modo 'a' (append) é útil:

```python
with open('dados.txt', 'a', encoding='utf-8') as arquivo:
    arquivo.write('Nova linha de dados\n')
```

Já para sobrescrever o conteúdo:

```python
with open('dados.txt', 'w', encoding='utf-8') as arquivo:
    arquivo.write('Dados atualizados\n')
```

#### 3. Atualizar Dados em Arquivos de Texto

Arquivos de texto não suportam edição direta; o procedimento padrão é ler, modificar na memória e reescrever:

```python
with open('dados.txt', 'r+', encoding='utf-8') as arquivo:
    linhas = arquivo.readlines()
    # modificando o conteúdo
    linhas[0] = 'Dados modificados\n'
    arquivo.seek(0)
    arquivo.writelines(linhas)
    arquivo.truncate()
```

#### 4. Exclusão de Dados (Removendo linhas específicas)

Sem suporte direto de exclusão, o método comum é criar um novo arquivo sem as linhas indesejadas:

```python
com inseto in linhas:
    if 'excluir' not in linha:
        linhas_filtradas.append(linha)

with open('dados_filtrados.txt', 'w', encoding='utf-8') as arquivo:
    arquivo.writelines(linhas_filtradas)
```

```python
cursor.execute('INSERT INTO Usuarios (nome, idade) VALUES (?, ?)', ('João', 30))
```

### 1. Programação Assíncrona em Python

A programação assíncrona permite que aplicações gerenciem múltiplas tarefas simultaneamente sem bloquear a execução, otimizando recursos e aumentando a performance, especialmente em aplicações I/O-bound como servidores web, clientes de rede, ou processos que lidam com muitas operações de entrada/saída.

#### 1.1 Introdução ao `asyncio`

O módulo `asyncio` fornece uma infraestrutura para escrever código assíncrono usando `async`/`await`.

**Exemplo básico:**

```python
import asyncio

async def saudacao(nome):
    print(f"Olá, {nome}!")
    await asyncio.sleep(1)
    print(f"Até logo, {nome}!")

async def main():
    tarefas = [
        saudacao("Maria"),
        saudacao("João"),
        saudacao("Ana")
    ]
    await asyncio.gather(*tarefas)

asyncio.run(main())
```

**Dicas práticas:**
- Use `asyncio.create_task()` para criar tarefas concorrentes.
- Utilize `asyncio.gather()` para aguardar múltiplas tarefas simultaneamente.
- Prefira `asyncio.run()` na entrada do programa para gerenciar o ciclo de evento.

#### 1.2 Exemplos de casos de uso reais
- Bots de chat
- Scrapers web que fazem várias requisições em paralelo
- Servidores de redes de alta performance

# Capítulo 5: Manipulação de Arquivos e Persistência de Dados

## 5.1 Introdução à Manipulação de Arquivos

Python oferece diversas maneiras de trabalhar com arquivos, permitindo leitura, escrita e manipulação de dados armazenados em disco. Trabalhar com arquivos é essencial para a automação de tarefas, armazenamento de dados e integrações com outros sistemas.

### 5.2 Abrindo e Fechando Arquivos

A função principal para manipulação de arquivos é `open()`, que retorna um objeto arquivo. A sintaxe básica:

```python
arquivo = open('nome_arquivo.txt', 'modo')
# operações de leitura/escrita
arquivo.close()
```

**Modos comuns:**
- `'r'` – leitura (padrão)
- `'w'` – escrita (sobrescreve o arquivo se existir)
- `'a'` – anexar (adiciona ao final do arquivo)
- `'b'` – modo binário (usado junto com os anteriores)

### 5.3 Gerenciador de Contexto (`with`)

Para garantir o fechamento automático do arquivo, use a sintaxe `with`:

```python
with open('dados.txt', 'w') as arquivo:
    arquivo.write('Olá, Mundo!\n')
```

Isso evita vazamentos de recursos mesmo em caso de erro.

## 5.6 Trabalhando com Arquivos CSV

O módulo `csv` facilita a leitura e escrita de arquivos delimitados por vírgulas, comuns em planilhas:

```python
import csv

# Escrita
```

## 1. Programação Assíncrona em Python

### 1.1 Introdução
A programação assíncrona permite que o Python execute operações I/O-bound de forma eficiente, evitando bloqueios e aumentando a performance de aplicações que lidam com múltiplas tarefas simultaneamente, como servidores web e bots.

### 1.2 O conceito de `async` e `await`
Python fornece a sintaxe `async def` para definir funções assíncronas e `await` para aguardar sua conclusão.

```python
import asyncio

async def tarefa_exemplo():
    print("Início da tarefa")
    await asyncio.sleep(2)
    print("Fim da tarefa")

# Executando a tarefa
asyncio.run(tarefa_exemplo())
```

### 1.3 Criando e gerenciando tarefas
Você pode criar múltiplas tarefas com `asyncio.create_task()` para executar tarefas concorrentes.

```python
async def tarefa1():
    await asyncio.sleep(1)
    print("Tarefa 1 concluída")

async def tarefa2():
    await asyncio.sleep(2)
    print("Tarefa 2 concluída")

async def main():
    t1 = asyncio.create_task(tarefa1())
    t2 = asyncio.create_task(tarefa2())
    await t1
    await t2

asyncio.run(main())
```

### 1.4 Boas práticas
- Use `asyncio.gather()` para executar tarefas em paralelo.
- Evite blocos de código síncrono que bloqueiem a event loop.
- Utilize bibliotecas assíncronas para operações de rede, banco de dados e outros I/O.

## Manipulação Avançada de Arquivos e Bancos de Dados em Python

### Manipulação de Arquivos: Leitura, Escrita, Atualização e Exclusão

Python oferece várias formas de manipular arquivos de forma eficiente e segura, usando as funções integradas e módulos especializados. Vamos explorar operações avançadas com exemplos práticos.

#### 1. Leitura de Arquivos (Modo 'rb', 'rt' com grandes volumes)

Para processar grandes volumes de dados, é recomendado ler o arquivo em partes, usando buffers ou leitura linha a linha:

```python
with open('large_file.txt', 'r', encoding='utf-8') as arquivo:
    for linha in arquivo:
        processa(linha)  # substitua pela sua lógica de processamento
```

#### 2. Escrita (Incluindo Append e sobrescrever)

Para evitar perder dados, o modo 'a' (append) é útil:

```python
with open('dados.txt', 'a', encoding='utf-8') as arquivo:
    arquivo.write('Nova linha de dados\n')
```

Já para sobrescrever o conteúdo:

```python
with open('dados.txt', 'w', encoding='utf-8') as arquivo:
    arquivo.write('Dados atualizados\n')
```

#### 3. Atualizar Dados em Arquivos de Texto

Arquivos de texto não suportam edição direta; o procedimento padrão é ler, modificar na memória e reescrever:

```python
with open('dados.txt', 'r+', encoding='utf-8') as arquivo:
    linhas = arquivo.readlines()
    # modificando o conteúdo
    linhas[0] = 'Dados modificados\n'
    arquivo.seek(0)
    arquivo.writelines(linhas)
    arquivo.truncate()
```

#### 4. Exclusão de Dados (Removendo linhas específicas)

Sem suporte direto de exclusão, o método comum é criar um novo arquivo sem as linhas indesejadas:

```python
com inseto in linhas:
    if 'excluir' not in linha:
        linhas_filtradas.append(linha)

with open('dados_filtrados.txt', 'w', encoding='utf-8') as arquivo:
    arquivo.writelines(linhas_filtradas)
```

## 1. Programação Assíncrona em Python

### 1.1 Introdução à Programação Assíncrona

A programação assíncrona permite que seu programa execute múltiplas tarefas de forma concorrente, sem bloquear o fluxo principal. É ideal para aplicações que envolvem operações de entrada/saída (I/O), como requisições de rede, acesso a banco de dados ou leitura/escrita de arquivos.

Python oferece suporte à programação assíncrona através do módulo `asyncio`, que permite escrever código assíncrono baseado em corrotinas.

### 1.2 Corrotinas, `async` e `await`

- **Corrotinas** são funções especiais que podem ser pausadas e retomadas, facilitando operações assíncronas.
- As palavras-chave **`async`** e **`await`** são usadas para definir e executar corrotinas.

**Exemplo básico:**

```python
import asyncio

async def tarefa(nome, tempo):
    print(f"Começando {nome}")
    await asyncio.sleep(tempo)
    print(f"Concluído {nome}")

async def main():
    await asyncio.gather(
        tarefa("Tarefa 1", 2),
        tarefa("Tarefa 2", 1),
    )

if __name__ == "__main__":
    asyncio.run(main())
```

**Dicas:**
- Use `asyncio.gather()` para rodar múltiplas tarefas em paralelo.
- Use `asyncio.create_task()` para agendar tarefas que podem rodar simultaneamente.

### 1.3 Ferramentas e Bibliotecas Recomendadas

- `asyncio`: biblioteca padrão para programação assíncrona.
- `aiohttp`: para requisições HTTP assíncronas.
- `aiomysql`, `aiopg`: para conexão assíncrona com bancos de dados MySQL e PostgreSQL, respectivamente.

```python
import asyncio

async def tarefa(nome, delay):

async def main():
    await asyncio.gather(
```

### Seção 1: Programação Assíncrona em Python

A programação assíncrona permite que seu programa realize várias tarefas ao mesmo tempo, sem bloquear o fluxo de execução principal. É especialmente útil para operações de entrada/saída, como chamadas de rede ou leitura de arquivos, onde esperar a resposta pode consumir tempo desnecessariamente.

#### 1.1 Introdução ao `asyncio`

O módulo `asyncio` fornece uma infraestrutura para escrever código assíncrono, permitindo criar tarefas concorrentes e gerenciar eventos de forma eficiente.

**Exemplo básico:**

```python
import asyncio

async def tarefa(nome, tempo):
    print(f"{nome} iniciado")
    await asyncio.sleep(tempo)
    print(f"{nome} concluído após {tempo} segundos")

async def main():
    await asyncio.gather(
        tarefa("Tarefa 1", 2),
        tarefa("Tarefa 2", 1),
        tarefa("Tarefa 3", 3)
    )

asyncio.run(main())
```

Esse exemplo executa várias tarefas concorrentes, otimizando o tempo total de execução.

#### 1.2 Criando e gerenciando tarefas assíncronas

- **`asyncio.create_task()`**: cria uma tarefa que roda em segundo plano.
- **`await`**: aguarda a conclusão de uma tarefa ou evento.

**Dicas:**
- Use `asyncio.gather()` para rodar múltiplas tarefas em paralelo.
- Use `asyncio.create_task()` para iniciar tarefas sem aguardar que terminem imediatamente, permitindo execução assíncrona.

#### 1.3 Melhores práticas
- Prefira funções `async` sempre que possível ao lidar com operações de I/O.
- Gerencie tarefas com cautela, cancelando ou aguardando adequadamente para evitar vazamentos de recursos.

### 4.1 Programação Orientada a Objetos (POO)

A POO é um paradigma que organiza o código em **classes** e **objetos**, facilitando a modularidade, reutilização e manutenção do software.

#### 4.1.1 Definição de Classes e Objetos

Uma classe é uma "forma" que define atributos (dados) e métodos (funções). Um objeto é uma instância dessa classe.

**Exemplo:**

```python
class Pessoa:
    def __init__(self, nome, idade):
        self.nome = nome
        self.idade = idade

    def apresentar(self):
        print(f"Olá, meu nome é {self.nome} e tenho {self.idade} anos.")

# Criando um objeto
p1 = Pessoa("Maria", 30)
p1.apresentar()
```

**Saída:**
```
Olá, meu nome é Maria e tenho 30 anos.
```

#### 4.1.2 Herança e Polimorfismo

A herança permite criar subclasses especializadas:

```python
class Estudante(Pessoa):
    def __init__(self, nome, idade, curso):
        super().__init__(nome, idade)
        self.curso = curso

    def apresentar(self):
        super().apresentar()
        print(f"Estudo no curso de {self.curso}")

estudante = Estudante("João", 22, "Engenharia")
estudante.apresentar()
```

## Seção Avançada do Manual de Python: Programação Assíncrona, Manipulação de Arquivos, Testes Automatizados e Otimizações de Desempenho

### 1. Programação Assíncrona em Python

A programação assíncrona permite executar múltiplas tarefas de forma eficiente, especialmente em operações I/O, como requisições de rede ou leitura/escrita de arquivos. Em Python, o módulo `asyncio` é fundamental para esse paradigma.

#### Exemplo básico com `asyncio`:
```python
import asyncio

async def tarefa(nome, tempo):
    print(f"Iniciando tarefa {nome}")
    await asyncio.sleep(tempo)
    print(f"Concluída tarefa {nome}")

async def main():
    await asyncio.gather(
        tarefa('A', 2),
        tarefa('B', 1),
        tarefa('C', 3)
    )

asyncio.run(main())
```
**Explicação:**  
- `async def` define corrotinas, que podem ser pausadas e retomadas.  
- `await asyncio.sleep()` simula uma operação assíncrona que pausa a execução sem bloquear o restante do programa.  
- `asyncio.gather()` executa várias tarefas em paralelo.

---

### 2. Manipulação de Arquivos de Forma Eficiente

A leitura e escrita de arquivos podem ser otimizadas usando o modo assíncrono com `aiofiles` para evitar bloqueios.

#### Leitura assíncrona de arquivo:
```python
import aiofiles
import asyncio

async def ler_arquivo(nome_arquivo):
    async with aiofiles.open(nome_arquivo, 'r') as f:
        conteudo = await f.read()
    print(conteudo)

asyncio.run(ler_arquivo('exemplo.txt'))
```

#### Escrita eficiente:
```python
async def escrever_arquivo(nome, conteudo):
    async with aiofiles.open(nome, 'w') as f:
        await f.write(conteudo)

asyncio.run(escrever_arquivo('saida.txt', 'Texto de exemplo'))
```
**Dica:** Use operações assíncronas para ler/gravar arquivos grandes ou em ambientes que exigem alta performance.
